Integration Guide

Import Sentinel Spec as a watsonx Orchestrate Skill.

Five steps to register the OpenAPI spec, configure authentication, and publish the governed compliance skill to your organization's catalog.

Prerequisites

You will need an IBM Cloud account with watsonx Orchestrate enabled, and the sentinel_spec_openapi.json file from the repository root.

For local evaluation, Docker Desktop and Ollama are required.

1
Export the OpenAPI spec

The file sentinel_spec_openapi.json is included in the repository root. It describes all endpoints including /evaluate, /evaluate/stream, /compliance/matrix, and /analytics/summary.

2
Navigate to Skill Studio

In your IBM Cloud account, open watsonx Orchestrate → Skill Studio → Add Custom Skill → Import from OpenAPI.

3
Upload the spec file

Select sentinel_spec_openapi.json from your repository. Skill Studio will parse all endpoints, request schemas, and response models automatically.

4
Configure authentication

Set the endpoint base URL to your deployed Code Engine app URL. Configure API key authentication using your watsonx API key secret.

5
Publish the skill

Assign access roles, set governance policies, and publish. The skill now appears in the Orchestrate catalog for all authorized team members.

MOCK_MODE Local Setup

terminal — Start infrastructure 
# 1. Start infrastructure
docker compose up -d

# 2. Pull the Granite model
ollama pull granite3.3:8b

# 3. Run with local adapters
MOCK_MODE=true uvicorn app:app --reload --port 8000

# 4. Verify
curl http://localhost:8000/health
terminal — Deploy to IBM Cloud Code Engine 
# Build and push
docker build -t sentinel-spec:latest .
docker tag sentinel-spec:latest <registry>/sentinel-spec:latest
docker push <registry>/sentinel-spec:latest

# Deploy to Code Engine
ibmcloud ce app create \
  --name sentinel-spec \
  --image <registry>/sentinel-spec:latest \
  --port 8000 \
  --env MOCK_MODE=false \
  --env WATSONX_API_KEY=@{secret:watsonx-api-key}
